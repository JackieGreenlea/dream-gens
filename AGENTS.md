# AGENTS.md

Build this app in small, reviewable steps.

Principles:
- Do not overengineer.
- Prefer simple, readable code.
- Keep the MVP tightly scoped.
- Do not add features that were not requested.
- Preserve the product goal: prompt -> structured world -> edit -> play.
- Use strict schemas for model outputs.
- Keep UI clean and calm.
- After each major task, explain what changed and why.
- If a task is too large, break it into smaller steps instead of improvising a huge architecture rewrite.
